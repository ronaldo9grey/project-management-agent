## 一、设计理念

### 传统模式 vs 新模式

| 维度 | 传统模式（人找数据） | 新模式（数据追人） |
|------|---------------------|-------------------|
| 发现问题 | 主动查询、定期检查 | 异常自动识别、主动推送 |
| 信息获取 | 报表、看板查询 | 智能摘要、关键洞察直达 |
| 决策支持 | 数据展示、人工分析 | AI分析、建议方案 |
| 时效性 | 滞后（日报/周报） | 实时监控、即时预警 |

### 核心原则

1. **主动推送**：重要信息自动触达，无需人工查询
2. **智能筛选**：AI 识别关键信息，过滤噪音
3. **个性化聚焦**：不同角色看到不同重点
4. **行动导向**：不只是展示问题，更给出建议

**数据追人体现**：

#### 模块二：预警中心

**预警类型**（共8类）：

| 优先级 | 类型 | 触发条件 | 严重程度 | 推送对象 |
|--------|------|---------|---------|---------|
| P1 | 🔴 延期预警 | 任务已过截止日期未完成 | 高 | 项目负责人 |
| P2 | 🟠 沉默预警 | 项目连续N天无日报/进度更新 | 中 | 项目负责人、管理层 |
| P3 | 🟡 成本预警 | 成本超支 >10% | 中 | 项目负责人、财务 |
| P4 | 🟡 进度预警 | 进度滞后 >20% | 中 | 项目负责人 |
| P5 | 🟡 资源冲突 | 同一人员同时在多个紧急任务 | 中 | HR、项目经理 |
| P6 | 🔵 节点预警 | 关键节点即将到期（3天内） | 低 | 项目负责人 |
| P7 | 🔵 里程碑漂移 | 关键节点日期变更≥2次 | 低 | 项目负责人 |
| P8 | 🔵 依赖阻塞 | 前置任务延期影响后续任务 | 低 | 项目负责人 |

**预警规则配置**（管理员可调整）：

```
延期预警阈值：
  - 延期1-3天：低风险
  - 延期4-7天：中风险
  - 延期>7天：高风险

沉默预警阈值：
  - 连续3天无更新：低风险
  - 连续5天无更新：中风险
  - 连续7天无更新：高风险

成本预警阈值：
  - 超支10%-20%：低风险
  - 超支20%-50%：中风险
  - 超支>50%：高风险
```

**预警规则展示**：
- 看板右上角「预警规则」按钮，点击查看完整规则
- 预警卡片显示触发原因和阈值
- 管理员可进入设置调整阈值

**数据追人体现**：
- 预警自动生成，无需人工检查
- 按严重程度分级显示
- 直接关联责任人

#### 模块三：项目健康度评分

**评分维度**（百分制）：

```
健康度 = 进度得分×0.4 + 成本得分×0.3 + 风险得分×0.3

进度得分 = 100 - max(0, 滞后天数×5) - 延期任务比例×20
成本得分 = 100 - max(0, 超支比例×2)
风险得分 = 100 - 高风险任务数×10 - 中风险任务数×5
```

**可视化**：
- 🟢 80-100分：健康
- 🟡 60-79分：需关注
- 🔴 0-59分：高风险

#### 模块四：个性化关注

**"我关注的项目"**：
- 我负责的项目
- 我参与的任务
- 我创建的日报关联的项目

**数据追人体现**：
- 不同用户看到不同内容
- 自动聚焦相关项目
- 减少信息噪音

## 五、推送方案

### 5.1 推送渠道

| 渠道 | 场景 | 实现方式 |
|------|------|---------|
| **飞书** | 工作通知（主要） | 已有飞书机器人 |

### 5.2 微信推送方案

**问题**：个人公众号（订阅号）功能受限，无模板消息权限

**推荐方案**：使用第三方推送服务

| 服务 | 免费额度 | 特点 |
|------|---------|------|
| **Server酱** | 5条/天 | 简单，微信服务号推送 |
| **PushPlus** | 200条/天 | 支持模板，企业微信 |
| **WxPusher** | 5000条/天 | 需用户关注公众号 |

**推荐：PushPlus**

```
实现流程：
1. 用户在设置页面绑定微信（扫码关注公众号）
2. 系统记录用户 openId
3. 预警触发时调用 PushPlus API 推送

推送效果：
- 微信服务号消息
- 支持富文本模板
- 支持点击跳转链接
```

**API示例**：

```python
import requests

def push_to_wechat(user_token: str, title: str, content: str):
    """通过 PushPlus 推送到微信"""
    url = "http://www.pushplus.plus/send"
    data = {
        "token": user_token,      # 用户在 PushPlus 获取的 token
        "title": title,
        "content": content,
        "template": "html"        # 支持 html/txt/json
    }
    requests.post(url, json=data)

# 使用
push_to_wechat(
    user_token="user_pushplus_token",
    title="项目预警：田林铝厂进度滞后",
    content="<b>项目</b>：田林铝厂<br><b>状态</b>：延期3天"
)
```

```python
async def get_health_trend(project_id: int, days: int = 30):
    """获取项目健康度趋势"""

    snapshots = db.query(ProjectHealthSnapshot)
        .filter(ProjectHealthSnapshot.project_id == project_id)
        .filter(ProjectHealthSnapshot.snapshot_date >= today - days)
        .order_by(ProjectHealthSnapshot.snapshot_date)
        .all()

    return {
        "project_id": project_id,
        "period": f"最近{days}天",
        "trend": [{
            "date": s.snapshot_date,
            "health_score": s.health_score,
            "progress_score": s.progress_score,
            "cost_score": s.cost_score,
            "risk_score": s.risk_score
        } for s in snapshots],
        "analysis": analyze_trend(snapshots)  # AI分析趋势
    }
```
