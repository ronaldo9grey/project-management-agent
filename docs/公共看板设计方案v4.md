# 公共看板设计方案 v4.0

> 基于用户反馈：样式优化 + 展示具体原因 + 按项目展示时间线 + 去除重复

---

## 一、核心改进点

| 问题 | 改进方案 |
|------|---------|
| 样式不好看 | 简化卡片，突出关键数字 |
| 只看到结果，看不到原因 | 点击展开具体项目/任务列表 |
| 时间线混乱 | 按项目分组展示，每个项目一条时间线 |
| 健康度重复 | 去掉健康度排行模块 |

---

## 五、交互设计

### 5.1 健康度分布

- 点击环形图区域 → 展开对应项目列表
- 点击项目名称 → 跳转到项目详情页

### 5.2 预警详情

- 默认显示高风险预警
- 点击展开/收起按钮查看更多
- 点击"标记已处理" → 处理预警

### 5.3 项目时间线

- 默认显示风险项目 + 关注项目（前5个）
- 折叠显示健康项目
- 点击项目名称 → 跳转到项目详情页

## 六、数据需求

### 6.1 时间线数据

需要从任务的 `start_date` 和 `end_date` 提取关键节点，并标记状态。

```typescript
interface ProjectTimeline {
  project_id: number
  project_name: string
  health_score: number
  nodes: {
    task_name: string
    date: string
    status: 'completed' | 'delayed' | 'risk' | 'ongoing' | 'pending'
  }[]
}
```

需要新增或修改：

## 七、对比总结

| 维度 | v3.0 | v4.0 |
|------|------|------|
| 健康度展示 | 环形图 + 排行榜 | 环形图 + 点击展开列表 |
| 预警展示 | 弹窗列表 | 页面内展开 + 具体原因 |
| 时间线 | 混合展示 | 按项目分组 |
| 信息深度 | 只有数字 | 数字 + 原因 + 责任人 |
| 去重 | 重复展示健康度 | 去掉排行榜 |
